# 유닉스 / 리눅스 소개 요약

## 1. 리눅스의 필요성

유닉스는 1970년대 초 AT&T 벨 연구소에서 개발된 이후 지속적으로 발전해 온 운영체제이다.  
현재는 PC, 서버, 스마트폰, 슈퍼컴퓨터 등 다양한 시스템에서 사용되는 핵심 운영체제가 되었다.

유닉스/리눅스 기반 운영체제의 예시는 다음과 같다.

- Android OS
- iOS
- Mac OS X
- Linux
- BSD Unix
- System V
- Solaris
- IBM AIX
- HP-UX
- Unicos

이처럼 유닉스 계열 운영체제는 현대 컴퓨팅 환경에서 중요한 기반 기술로 사용된다.

---

# 2. 유닉스의 설계 철학

유닉스는 다음과 같은 철학을 기반으로 설계되었다.

## 2.1 단순성 (Simplicity)

유닉스는 복잡한 시스템인 MULTICS에 대한 반성에서 출발하였다.  
필요한 최소한의 기능만 제공하고, 시스템 자원에 대해 일관된 관점을 제공하도록 설계되었다.

## 2.2 이식성 (Portability)

유닉스는 C 언어로 작성되어 다양한 하드웨어 환경으로 쉽게 이식할 수 있다.  
이러한 특징 덕분에 스마트폰, PC, 서버, 슈퍼컴퓨터 등 다양한 플랫폼에서 사용된다.

## 2.3 개방성 (Openness)

유닉스 계열 시스템은 소스 코드 공개와 같은 개방성을 기반으로 발전하였다.  
이로 인해 많은 개발자와 기업이 참여하여 지속적으로 발전할 수 있었다.

---

# 3. 유닉스의 특징

## 3.1 다중 사용자 (Multi-user)

여러 사용자가 하나의 시스템을 동시에 사용할 수 있다.

## 3.2 다중 프로세스 (Multi-process)

여러 프로그램을 동시에 실행할 수 있다.  
이를 통해 효율적인 자원 활용이 가능하다.

## 3.3 관리자 계정 (Superuser)

시스템 전체를 관리하는 특별한 사용자 계정이 존재한다.

## 3.4 쉘 프로그래밍

명령어와 유틸리티를 조합하여 자동화 프로그램을 작성할 수 있다.

## 3.5 뛰어난 네트워크 기능

유닉스는 초기부터 네트워크 환경을 고려하여 설계되었다.  
대표적인 네트워크 서비스는 다음과 같다.

- FTP
- Telnet
- WWW
- X Window

# 4. 유닉스 시스템 구조

유닉스 운영체제는 크게 다음과 같은 구조로 이루어진다.

## 4.1 운영체제 (Operating System)

컴퓨터의 하드웨어 자원을 관리하고 프로그램이 실행될 수 있는 환경을 제공한다.

## 4.2 커널 (Kernel)

운영체제의 핵심 부분이다.  
하드웨어를 직접 제어하고 시스템 자원을 관리한다.

## 4.3 시스템 호출 (System Call)

응용 프로그램이 커널의 기능을 사용할 수 있도록 제공되는 인터페이스이다.

## 4.4 쉘 (Shell)

사용자와 운영체제 사이의 인터페이스 역할을 한다.  
사용자가 입력한 명령어를 해석하여 운영체제에 전달한다.

---

# 5. 커널의 역할

커널은 시스템 자원을 관리하는 핵심 구성 요소이다.

## 5.1 프로세스 관리 (Process Management)

여러 프로그램이 동시에 실행될 수 있도록 CPU 스케줄링을 수행한다.

## 5.2 파일 관리 (File Management)

디스크에 저장된 파일과 파일 시스템을 관리한다.

## 5.3 메모리 관리 (Memory Management)

프로그램이 사용할 메모리를 효율적으로 할당하고 관리한다.

## 5.4 입출력 및 장치 관리 (I/O Management)

주변 장치와의 데이터 입출력을 관리한다.

## 5.5 통신 관리 (Communication Management)

프로세스 간 통신과 네트워크 통신을 관리한다.
