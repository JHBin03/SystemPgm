# 쉘 스크립트 기초 (Chapter 10 요약 및 실습)

## 1. 쉘 스크립트의 기본 구조
쉘 스크립트는 여러 명령어를 파일로 묶어 자동화하는 텍스트 파일입니다[cite: 1].

*   **Shebang (쉬뱅)**: 스크립트의 첫 줄에 `#!/bin/bash`를 기입하여 실행 환경을 명시합니다[cite: 1].
*   **주석 (Comments)**: `#` 기호 이후의 내용은 셸에서 무시하며 코드 설명을 위해 사용합니다[cite: 1].
*   **다중 명령 실행**: 한 줄에서 명령어들을 세미콜론(`;`)으로 구분하여 연속 실행할 수 있습니다[cite: 1].
    *   예: `echo "Hello" ; date`[cite: 8]

## 2. 메시지 출력 및 사용자 입력
*   **`echo` 명령어**: 터미널에 텍스트를 출력합니다[cite: 1].
    *   `-n` 옵션: 메시지 출력 후 줄 바꿈을 하지 않습니다[cite: 1].
*   **`read` 명령어**: 사용자로부터 입력을 받아 변수에 저장하며, `-p` 옵션으로 프롬프트를 띄울 수 있습니다[cite: 1].

## 3. 변수 사용 (Variables)
스크립트 내에서 데이터를 저장하고 참조할 때 사용합니다[cite: 1].

### 3.1 시스템 환경 변수
*   `$USER`: 현재 사용자 이름[cite: 4]
*   `$UID`: 사용자의 고유 ID 번호[cite: 4]
*   `$HOME`: 사용자의 홈 디렉터리[cite: 4]
*   `$PATH`: 명령어 검색 경로[cite: 4]

### 3.2 사용자 정의 변수
*   **할당**: `days=10` 처럼 `=` 기호 앞뒤에 공백 없이 할당합니다[cite: 5, 6].
*   **참조**: 변수명 앞에 `$`를 붙여 값을 불러옵니다[cite: 6].
    *   예: `value2=$value1`[cite: 6]

### 3.3 명령 치환 (Command Substitution)
명령어의 실행 결과를 변수에 저장할 때 백틱(`` ` ``) 또는 `$()`를 사용합니다[cite: 1].
*   예: `testing=$(date)` 또는 `testing=`date``[cite: 7, 14]

## 4. 데이터 리다이렉션 및 파이프
*   **출력 리다이렉션 (`>`, `>>`)**: 결과를 파일로 저장하거나 추가합니다[cite: 1].
*   **입력 리다이렉션 (`<`)**: 파일 내용을 명령어의 입력으로 전달합니다[cite: 1].
*   **파이프 (`|`)**: 명령어의 출력을 다음 명령어의 입력으로 즉시 전달합니다[cite: 1].

## 5. 수학 연산 (Math)
쉘은 기본적으로 문자열을 다루므로 산술 연산을 위해 특정 형식을 사용해야 합니다[cite: 1].

*   **`expr` 명령어**: 고전적인 정수 계산 방식이며 연산자 사이에 공백이 필수입니다[cite: 1].
    *   예: `var3=`expr $var2 / $var1``[cite: 14]
*   **대괄호 사용 (`$[]`)**: 괄호 내부에서 변수 앞의 `$`를 생략할 수 있으며 복잡한 수식 구성이 가능합니다[cite: 1].
    *   기본 연산: `var3=$[ $var1 + var2 ]`[cite: 12, 13]
    *   중첩 연산: $var4 = var1 \times (var2 - var3)$ 형식으로 작성 시 `$[$var1 * ($var2 - $var3)]` 처럼 기술합니다[cite: 15].
*   **`bc` 계산기**: 부동 소수점(실수) 연산 시 사용하며 `scale`로 소수점 자리를 지정합니다[cite: 1].
    *   예: `echo "scale=4; $var1 / $var2" | bc`[cite: 3]

## 6. 종료 상태 코드 (Exit Status)
모든 명령어는 실행 후 `$?` 변수에 종료 상태(0~255)를 남깁니다[cite: 1].

*   **`exit` 명령어**: 스크립트를 즉시 종료하고 특정 상태 코드를 반환합니다[cite: 1].
    *   연산 결과 반환: `exit $var3`을 사용하여 계산된 값을 상태 코드로 보낼 수 있습니다[cite: 12, 13].
*   **주요 코드**:
    *   `0`: 성공[cite: 1]
    *   `127`: 명령어를 찾을 수 없음[cite: 1]
    *   `126`: 실행 권한 없음[cite: 1]
