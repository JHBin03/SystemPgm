## 시스템 호출(System Calls)

운영체제의 핵심을 커널(Kernel)이라고 한다.
시스템 호출(System Call)은 응용 프로그램이 커널의 기능을 사용하기 위한 인터페이스이다.

응용 프로그램은 시스템 호출을 통해 운영체제에게 서비스를 요청한다.
즉, 사용자 프로그램 → 시스템 호출 → 커널 → 하드웨어 흐름으로 동작한다.

시스템 호출은 함수처럼 호출되지만 실제로는 커널 모드에서 실행된다.

---

## C 라이브러리 함수(Library Functions)

C 라이브러리 함수는 시스템 호출을 직접 사용하기 쉽게 감싸놓은 함수이다.

즉, 대부분의 C 함수는 내부적으로 시스템 호출을 사용한다.

예:

* printf() → 내부적으로 write() 시스템 호출 사용
* fopen() → 내부적으로 open() 시스템 호출 사용

ANSI C Standard Library는 이러한 함수들을 표준화하여 제공한다.

---

## 시스템 호출 vs 라이브러리 함수

| 구분     | 시스템 호출            | C 라이브러리 함수            |
| ------ | ----------------- | --------------------- |
| 실행 위치  | 커널 영역             | 사용자 영역                |
| 역할     | 커널 서비스 요청         | 시스템 호출을 감싸서 제공        |
| 사용 난이도 | 어려움               | 쉬움                    |
| 예      | open, read, write | fopen, fread, fprintf |

---

## 파일이 필요한 이유

프로그램에서 변수에 저장된 데이터는 프로그램이 종료되면 모두 사라진다.

데이터를 영구적으로 저장하기 위해 파일이 필요하다.

---

## 유닉스 파일 특징

유닉스 시스템에서는 모든 데이터를 파일로 취급한다.

* 모든 데이터는 연속된 바이트 형태로 저장된다.
* 키보드, 마우스, 장치도 파일로 간주된다.

---

## C 언어 파일 종류

### 텍스트 파일(Text File)

사람이 읽을 수 있는 문자 형태로 저장된 파일이다.

* 줄 끝(Line End)은 내부적으로 변환된다.
* 예: 메모장 파일

---

### 이진 파일(Binary File)

데이터를 그대로 바이트 형태로 저장하는 파일이다.

* 메모리에 저장된 값 그대로 저장 가능
* 사람이 직접 읽기 어려움

---

## 파일 입출력 과정

C 언어에서 파일 입출력은 다음 순서를 따른다.

1. 파일 열기: fopen()
2. 파일 입출력: fread, fwrite, fgetc, fputc 등
3. 파일 닫기: fclose()

---

## 파일 열기: fopen()

```c
FILE *fopen(const char *filename, const char *mode);
```

### 매개변수

* filename: 파일 이름
* mode: 파일 열기 모드

### 반환값

* 성공: FILE 포인터
* 실패: NULL

---

## FILE 구조체

FILE 구조체는 하나의 파일 스트림에 대한 정보를 담고 있다.

포함 내용:

* 버퍼 포인터
* 버퍼 크기
* 파일 상태 정보
* 파일 디스크립터

FILE 포인터는 fopen() 함수가 반환하며, 이후 모든 파일 입출력 함수에서 사용된다.

---

## 파일 디스크립터(File Descriptor)

파일 디스크립터는 운영체제가 파일을 식별하기 위해 사용하는 정수 값이다.

C 표준 라이브러리에서는 FILE 구조체 내부에서 파일 디스크립터를 관리한다.

---

## 표준 입출력 스트림

프로그램 실행 시 자동으로 열리는 스트림이다.

* stdin  : 표준 입력 (키보드)
* stdout : 표준 출력 (화면)
* stderr : 표준 오류 출력

```c
#include <stdio.h>
```

---

## 파일 닫기: fclose()

```c
int fclose(FILE *fp);
```

### 반환값

* 성공: 0
* 실패: EOF (-1)

파일을 사용한 후 반드시 닫아야 한다.

---

## 문자 단위 파일 입출력

### fgetc()

```c
int fgetc(FILE *fp);
```

* 파일에서 한 문자 읽기
* 파일 끝이면 EOF(-1) 반환

---

### fputc()

```c
int fputc(int c, FILE *fp);
```

* 파일에 한 문자 출력
* 성공 시 문자 반환
* 실패 시 EOF(-1) 반환

---

## 텍스트 파일 입출력 특징

* 문자 단위로 처리된다.
* 줄 끝은 내부적으로 변환된다.
* 사람이 읽을 수 있다.

---

## 이진 파일 입출력 특징

* 데이터를 그대로 바이트로 저장한다.
* 구조체, 변수 그대로 저장 가능하다.
* 빠르고 효율적이다.

---

## 핵심 정리

* 시스템 호출은 커널 기능을 사용하는 인터페이스이다.
* C 라이브러리 함수는 시스템 호출을 감싸서 제공한다.
* 파일은 데이터를 영구적으로 저장하기 위해 필요하다.
* 유닉스에서는 모든 것을 파일로 취급한다.
* 파일 입출력은 fopen → 입출력 → fclose 순서로 진행한다.
* FILE 구조체는 파일 스트림 정보를 담는다.
* 표준 입출력 스트림(stdin, stdout, stderr)은 자동으로 열린다.
